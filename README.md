# ミールログ — 写真でカロリー・PFC記録アプリ

写真を撮るだけでカロリーとPFC(たんぱく質・脂質・炭水化物)を記録できるPWA(ホーム画面に追加できるWebアプリ)。

## 機能

- 📷 **写真AI解析** — 食事の写真からメニュー名・カロリー・PFCを自動推定(Claude API・任意)
- ✍️ **手動入力 / マイメニュー** — APIキーなしでも完全に使える。よく食べるメニューはワンタップ登録
- 📊 **日別ダッシュボード** — 目標カロリーへの進捗、PFCバランス(エネルギー比率)
- 📈 **履歴** — 7/14/30日のカロリー推移・PFC比率グラフ・期間サマリー
- 💾 **データ管理** — JSONバックアップ / CSV書き出し(スプレッドシート用)/ 復元
- 📱 **PWA** — オフライン動作、ホーム画面追加でアプリのように使える

データはすべて端末内(ブラウザのIndexedDB/localStorage)に保存。外部送信は写真解析時のAnthropic APIのみ。

## スマホで使うには(デプロイ)

PWA(カメラ・ホーム画面追加)にはHTTPSが必要。無料の静的ホスティングに置くのが簡単:

### 方法A: Netlify Drop(最速・アカウント登録のみ)
1. https://app.netlify.com/drop を開く
2. `calorie-tracker` フォルダをドラッグ&ドロップ
3. 発行されたURLをスマホで開き「ホーム画面に追加」

### 方法B: GitHub Pages
1. GitHubにリポジトリを作り、このフォルダの中身をpush
2. Settings → Pages → Branch: main で公開
3. `https://<ユーザー名>.github.io/<リポジトリ名>/` をスマホで開く

※ URLを知っている人は誰でも開けますが、データは各端末内にしか保存されないので他人に記録が見えることはありません。

## AI写真解析のセットアップ(任意)

1. https://console.anthropic.com/ でアカウント作成(claude.aiのサブスクとは別の従量課金)
2. Billingでクレジットをチャージ(最低$5。プリペイド式なので使いすぎない)
3. API Keys でキーを発行
4. アプリの「設定」タブにキーを貼り付け、モデルを選択

### 料金の目安(1食の写真解析あたり)

| モデル | 精度 | 目安 |
|---|---|---|
| Opus 4.8 | 最高 | 約3〜5円/食(1日3食で月300〜450円) |
| Sonnet 5 | 高 | 約1〜2円/食 |
| Haiku 4.5 | 十分実用的 | 約0.5円/食(1日3食で月約50円) |

## ローカルで試す

```sh
cd calorie-tracker
python3 -m http.server 8742
# → http://localhost:8742
```

## 機種変更・バックアップ

設定 → 「バックアップを書き出す(JSON)」でファイル保存 → 新端末で「バックアップを読み込む」。APIキーはバックアップに含まれないので再設定してください。
